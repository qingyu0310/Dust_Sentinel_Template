# 学习路线重设计：以你的能力层级为中心，而不是以项目为中心

## 1. 这份文档现在在解决什么问题

这份文档的目标已经变了。

它不再是：

```text
基于这个底盘项目，下一步该做什么
```

而是：

```text
从这个项目反推出你现在大概处在什么能力层级，
然后重新设计一条更通用、更长期的学习路线
```

也就是说，这个项目在本文里只承担一个作用：

```text
作为能力样本，帮助判断你目前已经会什么、不会什么、下一步该补什么
```

它不是这条学习路线的中心。

---

## 2. 先判断你现在大概是什么水平

如果只看你这个项目能反映出来的能力，我对你当前水平的判断是：

### 2.1 你已经明显不是“初学者”

你已经不是那种刚学会 PID、刚学会单片机、刚会写一个 demo 的阶段了。

从这个项目能看出来，你已经具备下面这些能力：

1. 你能从 0 到 1 搭一个完整系统，而不是只能改局部模块。
2. 你能把控制问题拆成线程、数据流、控制器、约束层、通信层。
3. 你能把代码、文档、实验想法、训练脚本同时往前推进。
4. 你不是只会“调参数”，而是在主动思考结构设计。
5. 你已经开始碰“控制 + 实时系统 + ML 部署”的交叉问题。

这说明你的起点不是“怎么入门”，而是：

```text
已经能独立做出系统，但还在从“能做出来”走向“做得更深、更稳、更通用”
```

### 2.2 你更像哪一类工程师

如果用比较粗的分类，你现在更接近：

- 会独立做嵌入式系统的工程型开发者
- 会自己搭控制链路的应用型控制工程师
- 正在往系统型工程师和算法型工程师的交叉地带走

你还不只是“写固件的人”，也还不只是“调模型的人”。

你现在最有价值的成长方向，是把这几个能力打通：

- 控制理论
- 实时软件架构
- 实验与辨识方法
- 学习型控制
- 工程抽象能力

### 2.3 你当前最强的地方

这个项目反映出来的强项，不是“理论特别完整”，而是“能把事情搭起来并且往前推进”。

你的明显优势是：

1. 系统拆分能力
   - 你已经会把目标、反馈、控制、功率约束、输出链路分层。

2. 工程推进能力
   - 你不是停在想法上，而是已经写出线程、控制器、训练脚本和文档。

3. 问题意识
   - 你已经在思考真机抖动、时序、功率限制、训练和部署之间的关系。

4. 跨域兴趣和执行力
   - 很多人只会停留在控制或嵌入式某一块，你已经在往交叉方向试探。

### 2.4 你当前最该补的短板

你现在的短板，不是“完全不会”，而是“已经碰到上层难题，但底层方法论还不够系统”。

我会把短板归成 5 类：

1. 理论系统化不够
   - 你已经会做很多东西，但控制、建模、离散系统、优化这些知识还需要从“会用”升级到“成体系”。

2. 实验方法还不够标准化
   - 你已经有很强的工程直觉，但系统辨识、数据采集、对比实验、误差分析还需要更科研化。

3. 机器学习链路还偏早期
   - 你已经开始动训练脚本和 TFLM，但训练、验证、量化、部署一致性还没有形成完整套路。

4. 通用计算机基础还可以再上一个台阶
   - 你现在解决问题更偏工程实战，后续如果想走得更远，操作系统、编译、数值计算、软件设计会越来越重要。

5. 抽象与迁移能力还有升级空间
   - 你已经能把一个项目做出来，下一步要练的是“这个项目背后的方法能不能迁移到别的系统”。

## 3. 这条学习路线的设计原则

既然项目只是能力样本，那学习路线就不应该围着“这一个项目接下来怎么改”来排，而应该围着“你这个人下一阶段怎么升级”来排。

我会按下面 4 个原则设计路线。

### 3.1 不是补最弱的，而是补最能产生跃迁的

学习路线不是哪里不会补哪里，而是要优先补那些一旦补上，就会同时带动很多能力的知识。

对你来说，最能产生跃迁的不是某个单独 API，而是：

- 离散控制与系统理论
- 系统辨识与实验设计
- 实时系统与软件架构
- 数值建模与学习型控制

### 3.2 从“会做项目”升级到“会做可迁移的方法”

你已经证明了自己能做项目。

下一步的目标应该从：

```text
把这个项目做出来
```

升级成：

```text
把这类问题的方法论做出来
```

### 3.3 理论、工程、实验三条线并行

你这种类型的人，如果只学理论会空，只有工程会卡，只有 ML 会飘。

最好的方式是三条线同时推进：

1. 理论线：让你知道为什么。
2. 工程线：让你知道怎么稳定落地。
3. 实验线：让你知道怎么验证自己是不是对的。

### 3.4 以 12 到 18 个月为尺度看成长

如果只按一两周排，你会很容易陷入“眼前要不要改某个模块”的局部视角。

这次路线更适合用下面这个尺度来看：

- 3 个月：补齐基础短板
- 6 个月：形成完整方法
- 12 个月：建立自己的技术主线
- 18 个月：具备向更高层次迁移的能力

如果一定要用一句话概括你现在最适合走的方向，我会这样说：

```text
你最适合走“控制系统 + 嵌入式实时系统 + 学习型控制落地”这条复合路线

```

## 5. 新的学习路线总览

我建议把未来 12 到 18 个月的学习分成 5 条主线。

### 主线 A：控制与系统理论

目标：

- 把现在依赖工程直觉的很多做法，升级成可解释、可推导、可迁移的理论能力。

### 主线 B：实验设计与系统辨识

目标：

- 学会用更标准的方法描述对象，而不是只靠试错和经验判断。

### 主线 C：实时软件与系统工程

目标：

- 从“会写线程和模块”升级成“能稳定驾驭复杂实时系统”。

### 主线 D：数值计算、ML 与 TinyML

目标：

- 把学习型控制这条线补完整，不只是会搭一个小模型。

### 主线 E：抽象、表达与技术方法论

目标：

- 从“能做事”升级成“能总结规律、建立框架、迁移到新问题”。

## 6. 第一阶段：未来 0 到 3 个月该重点补什么

这一阶段的重点不是扩展领域，而是把底层能力打扎实。

### 6.1 第一优先级：离散控制与系统分析

这是你现在最值得补的第一块。

原因很简单：

- 你已经在做闭环控制系统了。
- 你已经碰到真实系统中的延迟、噪声、约束和时序。
- 你后面想做 NN 替代控制，必须先知道传统控制哪里强、哪里弱。

这一阶段要学的核心内容：

1. 连续系统与离散系统
2. 采样、保持、延迟
3. 阶跃响应、频域直觉、稳定性
4. 串级控制
5. 饱和、anti-windup、滤波
6. 状态空间基础

推荐资料：

- MIT 16.30 Feedback Control Systems
  - https://ocw.mit.edu/courses/16-30-feedback-control-systems-fall-2010/resources/lecture-notes/
- Underactuated Robotics
  - https://underactuated.mit.edu/
- Brian Douglas 的控制系统视频
  - https://www.youtube.com/@BrianBDouglas

这一阶段的目标不是“全学完”，而是做到：

- 你能用更正式的语言解释自己现在代码里的控制结构。
- 你能区分哪些问题是参数问题，哪些是结构问题，哪些是约束问题。

### 6.2 第二优先级：系统辨识与实验设计

你已经具备很强的动手能力，下一步要补的是“如何更科学地认识系统”。

要学的核心内容：

1. 输入输出实验设计
2. 阶跃、PRBS、扫频等激励方式
3. 简化模型拟合
4. 参数可辨识性
5. 误差来源拆解
6. 如何设计对照实验

推荐资料：

- Underactuated: System Identification
  - https://underactuated.mit.edu/sysid.html
- MathWorks System Identification Workflow
  - https://www.mathworks.com/help/ident/gs/system-identification-workflow.html

阶段目标：

- 你能独立设计一套数据采集方案。
- 你能把“真机为什么抖”拆成更明确的候选原因。

### 6.3 第三优先级：实时系统基础再系统化

你已经会写实时线程，但下一阶段要从“会用”升级到“懂调度、懂抖动、懂边界”。

要学的核心内容：

1. 线程调度与优先级
2. 定时与抖动
3. ISR 与线程边界
4. 消息队列、发布订阅、共享内存
5. 浮点、日志、锁对实时性的影响
6. 观测与 profiling

推荐资料：

- Zephyr Kernel Services
  - https://docs.zephyrproject.org/latest/kernel/services/
- Zephyr zbus
  - https://docs.zephyrproject.org/latest/services/zbus/index.html

阶段目标：

- 你能系统解释实时系统里“偶发抖动”的来源。
- 你能设计更可靠的观测手段，而不是只靠打印。

## 7. 第二阶段：未来 3 到 6 个月该升级什么

这阶段的核心，是把你从“做嵌入式控制项目的人”往“做智能控制系统的人”推进。

### 7.1 数值建模与 ML 训练流程

这部分不是为了让你转成纯 ML，而是为了让你把学习型控制真正做完整。

要学的核心内容：

1. 回归任务训练流程
2. 训练集/验证集/测试集设计
3. 特征工程与归一化
4. 误差分析与可视化
5. 模型导出与一致性验证
6. 小模型设计原则

推荐资料：

- TensorFlow Keras regression tutorial
  - https://www.tensorflow.org/tutorials/keras/regression
- Keras structured data examples
  - https://keras.io/examples/structured_data/
- TensorFlow training with built-in methods
  - https://www.tensorflow.org/guide/keras/training_with_built_in_methods

阶段目标：

- 你能把一个监督学习问题从数据到部署跑成完整闭环。
- 你能明确地说出模型失败是因为数据、特征、结构还是部署失配。

### 7.2 TinyML / TFLM 工程化能力

你未来如果想在这条路上做出自己的特色，这块非常关键。

要学的核心内容：

1. TFLM 模型加载
2. 算子约束
3. tensor arena
4. 推理耗时
5. 定点量化
6. 板端与 PC 一致性

推荐资料：

- TensorFlow Lite for Microcontrollers
  - https://www.tensorflow.org/lite/microcontrollers/overview
- TensorFlow post-training quantization
  - https://www.tensorflow.org/model_optimization/guide/quantization/post_training
- TensorFlow quantization aware training
  - https://www.tensorflow.org/model_optimization/guide/quantization/training

阶段目标：

- 你能独立把一个小模型部署到板端并验证。
- 你能判断一个模型“不适合上板”是因为结构问题还是部署问题。

### 7.3 软件设计能力再升级

这阶段你应该开始系统补一点更通用的软件基础。

建议学习：

1. 设计接口而不是只写模块
2. 状态机与事件驱动
3. 配置系统设计
4. 日志、监控、调试接口设计
5. 测试思维
6. 最基本的性能分析方法

推荐资料：

- A Philosophy of Software Design
- Designing Data-Intensive Applications
  - 不一定全部吃完，但很适合培养系统设计的视角

阶段目标：

- 你开始更自然地思考“如何设计长期可扩展的系统”，而不只是“这次先跑起来”。

## 8. 第三阶段：未来 6 到 12 个月该建立什么能力

这一阶段要开始建立你自己的技术方向，而不是单点补课。

### 8.1 方向一：学习型控制

如果你想继续沿现在的兴趣深入，这是最自然的方向。

建议重点学：

1. 模型学习与控制
2. imitation learning
3. safe RL
4. residual learning
5. model predictive control 的基本思想
6. constraints-aware control

这条线的关键不是“RL 很酷”，而是：

```text
怎样把学习方法纳入一个有安全边界的真实控制系统
```

### 8.2 方向二：高可靠嵌入式控制系统

如果你未来更偏工程系统，这条线也很强。

建议重点学：

1. 实时调试与性能分析
2. 驱动与总线抽象
3. 任务编排与容错
4. 标定系统
5. 数据回放系统
6. 线上回退与保护机制

这条线的价值是：

```text
你会变成那种能把复杂系统做稳的人
```

### 8.3 方向三：控制与算法的桥梁型角色

这是我觉得你很有潜力走的一条路。

你不必变成纯理论研究者，也不必变成纯软件架构师。

你可以逐步把自己变成这种人：

```text
既能理解真实系统约束，
又能把控制和学习方法真正落到硬件上
```

这种能力在很多团队里都很稀缺。

## 9. 第四阶段：未来 12 到 18 个月该追求什么

这阶段就不要再只看“学了哪些知识点”，而要看你是不是建立了自己的主线。

我建议你追求下面 3 个结果。

### 9.1 结果一：形成自己的方法论

你需要逐步把自己的做法总结成可复用套路，比如：

- 如何拆实时控制系统
- 如何做数据采集与辨识
- 如何让 NN 安全接管局部控制器
- 如何做灰度与回退

当你能把这些说清楚、写清楚、复用出来时，你就不只是“做过一个项目”，而是开始拥有自己的技术方法。

### 9.2 结果二：具备跨项目迁移能力

这很重要。

真正的成长不是：

```text
我把这个底盘项目做得很熟
```

而是：

```text
我把这个项目里抽出来的方法，放到别的机器人、别的电机系统、别的嵌入式平台上，仍然能成立
```

### 9.3 结果三：开始形成“研究型工程师”的气质

这里的“研究型”不是指读很多论文，而是指你开始具备下面这些习惯：

1. 提出假设
2. 设计实验
3. 控制变量
4. 记录结论
5. 复盘失败
6. 用证据而不是感觉做判断

这会让你后面无论走工程还是算法，成长速度都明显提升。

学习路线不是全都学，而是有意识地延后一些东西。

### 10.1 纯前端、纯产品、纯业务型软件能力

除非你接下来明确要转方向，否则这些不是你当前成长的主线。

### 10.2 过深的 Linux 应用栈

Linux 当然值得学，但你现在更值得先学的是：

- 实时系统
- 控制系统
- 软件架构
- 数值和实验方法

而不是先一头扎进复杂用户态框架。

### 10.3 太早追求复杂网络结构

你现在更需要：

- 好问题定义
- 好数据
- 好实验
- 好部署

而不是：

- 更深的网络
- 更复杂的 RL 框架
- 更多流行 trick
