# 底盘 NN 控制 - 项目总结

## 目标

用神经网络替代底盘行进电机（drive motor）的 PID 控制，实现速度环直出电流。

### 具体功能需求

1. **NN 替代两级 PID** — 去掉 `drive_velocity` 和 `drive_torque` 两个 PID，NN 直接输出电流值到 CAN 帧
2. **速度跟踪** — 根据 `[目标速度, 当前速度, 转矩]` 输出电流，稳态误差趋近 0
3. **抗扰动** — 外力扰动（推、阻力变化）时能自动补偿，不掉速度
4. **平滑控制** — 输出电流不突变，电机不振荡
5. **部署方式** — 训练好的 RL 策略导出 TFLite 模型 → 编译进 Zephyr 固件 → 板子上运行

### NN 接口设计

```
输入（3维）：[目标速度(m/s), 当前速度(m/s), 转矩(Nm)]
输出（1维）：[控制电流(A)]，tanh 激活，范围 [-1,1]，板子上乘以 20 得到实际安培
```

### 控制流（目标）

```
ReadRemote() → UpdateTarget() → [NN(目标速度, 当前速度, 转矩) → 电流] → PowerAlloc() → FramePublish()
```

### 关键约束

- 控制周期 1ms（与现有 PID 周期一致）
- 推理耗时 < 0.5ms（600MHz RISC-V 上实测 ~70µs）
- 不依赖遥控器以外的外部输入

## 硬件

| 项目 | 规格 |
|------|------|
| MCU | STM32F407IGH6 (Cortex-M4) |

| 底盘类型 | 双舵轮（2轮，前后布局） |
| 通信 | CAN 总线，1ms 控制周期 |
| 传感器 | 电机反馈：角度、速度、转矩（通过 CAN 读取） |

## 当前控制架构

### 控制流（每 1ms）

```
ReadRemote() → UpdateTarget() → ControlCalculate() → PowerAlloc() → FramePublish()
```

### 行进电机 PID（每轮）

```
速度环（P-only, Kp=5.0）→ 转矩环（P-only, Kp=0.5）→ 电流 → CAN
```

所有 PID 当前为 P-only（Ki=0, Kd=0），速度环输出转矩参考，转矩环输出电流。

### 电机参数

| 参数 | 值 | 来源 |
|------|----|------|
| kTorqueK (Kt) | 0.3 Nm/A | C620 手册 |
| 最大电流 | 20A | 硬件限幅 |
| 电流缩放 | 16384 / 20 | CAN 帧编码 |
| 最大速度 | 0.5 m/s | 遥控器上限 |

### 系统辨识结果（从真机数据）

```
J = 0.028（转动惯量）
B = 0.123（摩擦系数）
```

## TFLM 集成状态

- TFLite Micro 已集成到 Zephyr 工程
- Python 训练管线已就绪（从真机数据系统辨识 → 仿真训练 → TFLite 导出 → C 数组）
- RL 训练已跑通（PPO，200k 步），但策略在真机上振荡

## 已尝试的 NN 方案

1. **模仿学习（supervised）** — 用真机 PID 数据训练，NN 学的是烂 PID
2. **仿真训练 + 部署** — PC 仿真电机训 RL，真机上振荡
3. **Domain randomization** — 训练时随机化 J/B，效果有限

## 卡住的问题

仿真训练的策略在真机上振荡，根本原因：
- 仿真太理想，RL 学到的是"完美电机"的策略
- 真机有延迟、摩擦、惯量，RL 没学透鲁棒的控制策略
- 需要正确的 RL 训练框架（类似 PX4 的 Aerial Gym）
